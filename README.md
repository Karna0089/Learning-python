# 🐍 Learning Python: My Coding Journey

Welcome to my Python learning repository! This is a centralized place where I am uploading all the scripts and mini-games I build as I master the core basics of Python.

---

## 🗡️ Project 1: Text-Based RPG Shop & Inventory System (`hello.py`)
This is a beginner-friendly, text-based game script where a player can interact with a shopkeeper, manage their gold, and buy or sell fantasy items.

### Features
* **Interactive Menu:** A continuous game loop that lets the player choose their actions.
* **Dynamic Buying & Selling:** Players can buy items if they have enough gold, and sell items back to the shop for 70% of the original value.
* **Shopkeeper AI:** The shopkeeper has a randomized "mood" generated by the `random` module, meaning they won't always agree to buy your items!
* **Inventory Management:** Tracks the player's current coin balance and item quantities using Python dictionaries.

## ⚔️ Project 2: The Monster Arena (`arena.py`)
A text-based, turn-based combat system where the player battles a wild Goblin. It features dynamic combat mechanics, inventory management, and custom story elements.

### Features
* **Dynamic Combat:** Players choose to Attack, Heal, or Run, while the Goblin automatically counter-attacks.
* **The "Tackle" Mechanic:** The goblin has a 10% chance to dodge normal attacks.
* **Critical Hits:** A 20% chance to land a critical hit for double damage!
* **Dynamic Endings & Loot:** Running away has a 50/50 survival chance, and winning drops a random amount of gold.

## 🧠 Core Concepts Mastered
Across these projects, I have tackled and learned:
* Using **Dictionaries** (`.get()`, `.update()`, and key assignments) to manage game states.
* Writing `while True` loops and controlling the flow with `break`.
* **The D.R.Y. Principle:** Learning to modularize code using functions and passing arguments (like damage multipliers) to avoid repeating logic.
* Input sanitization (using `.lower()`) and handling `NoneType` errors.
* Guarding against logic bugs (like preventing players from "over-healing" past their max HP).
